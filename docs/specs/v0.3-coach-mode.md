# kerokero v0.3 — Coach Mode

## コンセプト

**「会話エンジンを作らない。採点エンジンに徹する」**

kerokero は *Examiner*（会話相手）ではなく **Coach**（コーチ）として再定義する。会話相手は外部に外注し、kerokero は録音を受け取って **厳密採点 + 学習ループへの還流** を担う。

### なぜ撤退するか
- ChatGPT Advanced Voice (gpt-realtime) は音声→音声ネイティブで往復 ~320ms
- Gemini Live も ~600ms-1s
- 段組み (VAD → STT → LLM → TTS) は構造的にこのレンジに勝てない
- Mac Studio + Kokoro MLX 実測（[scratch/localtts/RESULTS.md](../../../scratch/localtts/RESULTS.md)）: 80文字英文の同期合成だけで 1,478ms
- 会話品質 (発音情報・被せ・自然さ) も音声ネイティブモデルが上

→ 速度・自然さでは勝てない。**勝てる軸に絞る**。

### 差別化軸（v0.3 が担保すべき5つ）
1. **採点の厳密さ・透明性** — Band 根拠を構造化 JSON で
2. **オフライン/プライベート** — Whisper local 継続、音声・ログは手元
3. **カスタマイズ性** — プロファイル切替、独自 rubric、IELTS 以外の文脈
4. **Unix 哲学** — CLI、ローカル JSON、grep 可能、他ツールと連結
5. **コスト** — realtime API 課金を回避（長時間練習向き）

### 永久封印するもの
- ローカル Examiner 会話モード（barge-in / VAD / 双方向ストリーミング）
- Kokoro MLX 統合
- Silero VAD 統合
- 低遅延 TTS 一般
- **`kerokero drill` サブコマンド** — ShadowMaster Tab 3 (瞬間英作文) が同機能を既にカバー済みのため不要

## ShadowMaster (iOS) との役割分担

ShadowMaster v1.2 は3タブ構成に進化し、Tab 2 (Speaking Practice) は kerokero test mode と機能が重なる。並走させ、場面で使い分ける：

| ツール | 役割 | 場面 |
|---|---|---|
| **kerokero (Mac CLI)** | 厳密採点 / 任意音声ファイル解析 / TLR後の振り返り | 机/夜 |
| **ShadowMaster Tab 1** | 反復シャドーイング (kerokero analyze の弱点フレーズ素材を取り込む) | 歩き |
| **ShadowMaster Tab 2** | iPhone 単体での録音+即時採点 | 移動/外出先 |
| **ShadowMaster Tab 3** | 瞬間英作文 (226文プリセット + Claude生成) | スキマ時間 |

### 連携: kerokero analyze → ShadowMaster Tab 1

`export_shadow_drills()` は ShadowMaster の `SimpleJSONImporter` / `BulkImportView` がそのまま読める形式で出力する：

```json
[
  {"text": "I produced documentaries for NHK including...", "translation": "", "accent": "en-GB"}
]
```

- デフォルト accent は `en-GB`（Tab 3 と統一）
- ファイル: `~/.kerokero/sessions/<ts>_drills.json`
- 取り込み: ShadowMaster → Import → JSON ファイル選択

iOS 録音を Mac 側 inbox に流し込む経路は v0.4 以降で検討（Coach Mode ループが手動で1週間回ってから）。

## 機能仕様

### 新サブコマンド

```bash
# 1. 録音解析モード（v0.3 の本丸）
kerokero analyze --audio recording.m4a [--profile sales_english] [--topic "..."]
  → Whisper 文字起こし
  → プロファイル別評価（プロファイル無し時は IELTS 評価にフォールバック）
  → 構造化 JSON で band/rubric 根拠/弱点フレーズ/shadow drills を出力
  → ~/.kerokero/sessions/<ts>_analyze.json に保存

# 2. ドリル生成モード（Week 2）
kerokero drill --from-session latest --count 20

# 3. 進捗トラッキング（Week 2-3）
kerokero progress --weeks 4

# 4. Examiner ラッパー（条件付き・Week 3+）
kerokero examiner --backend gemini-live --profile ielts_speaking
```

### プロファイル仕様

`~/.kerokero/profiles/<name>.toml` または `<repo>/profiles/<name>.toml` に配置。
TOML を採用（kerokero 既存依存 `tomllib` で読めるため。spec 草案では YAML だったが、依存を増やさないための変更）。

```toml
name = "DELAX Sales English"
description = "Documentary production sales conversation"
focus = "sales"  # "sales" | "ielts" | "freeform"
display_lang = "ja"

[[rubric]]
key = "clarity_of_value_prop"
label_en = "Clarity of value proposition"
label_ja = "営業価値の伝達"

[[rubric]]
key = "production_credibility"
label_en = "Production credibility"
label_ja = "制作実績の説得力"

# ...

target_phrases = [
  "I produced documentaries for NHK including...",
  "We can shoot in Europe with full crew...",
]

weak_areas_focus = ["articles", "tense_switching", "output_volume"]
```

### 評価 JSON スキーマ（profile 経由時）

```json
{
  "overall_score": 7.0,
  "rubric_scores": { "clarity_of_value_prop": 7, "...": 6 },
  "strengths": ["..."],
  "weaknesses": ["..."],
  "specific_improvements": [
    {"original": "...", "improved": "...", "reason": "..."}
  ],
  "model_answer_en": "...",
  "examiner_comment": "...",
  "target_phrases_used": ["..."],
  "target_phrases_missed": ["..."],
  "shadow_drills": ["..."]
}
```

IELTS 経路 (`evaluate_test`) は既存スキーマを維持する。
