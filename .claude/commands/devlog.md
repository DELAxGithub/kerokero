今日の開発日誌を書く/更新する。

## 手順

1. 今日の日付を確認する（YYYY-MM-DD）
2. `devlog/{YYYY-MM-DD}.md` が既にあれば読み込む。なければ新規作成
3. 今回のセッションの作業を把握する:
   - git log --oneline（今日のコミット）
   - 会話内容から開発した内容
4. devlog を書く/更新する（下記フォーマット）
5. `devlog/README.md` のログ表を更新する（日付・Band・主な開発）
   - Day番号はREADME.mdの既存行から推算する（Day 1 = 03-12）

## devlog フォーマット

```markdown
# Day {N} — {YYYY-MM-DD}

## 開発（3行まとめ）
- 変更1
- 変更2
- 変更3

## 英語の練習

<!-- 練習ログはここに追記。まだ練習していない場合はこのコメントを残す -->
```

## ルール
- 「開発」は3行以内。非エンジニアにもわかる粒度
- 「英語の練習」セクションが既にある場合は消さない（追記のみ）
- 練習ログがまだない場合はコメントだけ残す。捏造しない
- devlog以外のファイルは変更しない
- 日本語で書く
